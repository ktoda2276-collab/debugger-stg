# 要件定義書：『（仮）DEBUGGER』

## 1. コンセプト

**「世界はバグに侵食された。アンチウイルスAIが、機体に宿って空からデバッグする。」**

プレイヤーはAI型アンチウイルスプログラムが憑依した戦闘機を操り、glitch化した世界をステージごとに修復していく王道2D横スクロールSTG。世界観は「壊れたゲーム空間そのもの」であり、敵・ステージ・演出すべてがglitch表現で統一される。

## 2. ターゲット & プレイ体験

- **ターゲット層**：カジュアル層、ブラウザでサクッと遊びたい人、レトロ／glitch美術の愛好者
- **1プレイ時間**：1ステージ3〜5分、全クリア30〜45分想定
- **プレイ感**：誰でも触れて、ボス戦と世界観演出で「お、なんかすごいの見た」と思える体験

## 3. プラットフォーム & 技術要件

- **形式**：HTML5ブラウザゲーム（PC/モバイル両対応が望ましい）
- **実装スタック**：Phaser 3 + Vanilla JS + Vite
- **対応操作**：キーボード（矢印+Z/X/C）、ゲームパッド対応、タッチ操作（モバイル）
- **保存**：LocalStorageでハイスコアと進行ステージのみ保存

## 4. ゲームシステム

### 4.1 操作
- 移動：8方向
- ショット：通常弾（押しっぱなし連射）
- ボンバー／必殺技：画面全体のバグを一掃する「強制デバッグ」（ストック制、各ステージ開始時に補充）

### 4.2 体力
- ライフゲージ制（HP3〜5マス相当）
- 被弾でゲージ減少、ステージ中にHP回復アイテム出現
- ゲージ0でゲームオーバー（ステージ最初orチェックポイントから再開）
- コンティニュー回数は無制限（カジュアル方針）

### 4.3 進行
- 面クリア式リニア進行（ステージセレクトなし）
- ステージ数：5〜8面 + 最終ボス
- 各面の構成：道中（2〜3パート）→ 中ボス → 道中 → ステージボス

### 4.4 スコア
- スコア表示のみ（実績・チャレンジは入れない）
- ハイスコアはLocalStorageに保存
- 加点要素：撃破、コンボ（連続撃破）、ノーミスボーナス、残りボンバー

## 5. 世界観 & ビジュアル

### 5.1 世界観設定
- 舞台：かつて正常に動いていた「世界」が、未知のバグに侵食されている
- 主人公：アンチウイルスAI「（仮称）」が、最後のクリーンな機体に宿って起動
- 各ステージ＝侵食された領域（メモリ領域・ファイルシステム・ネットワーク層など、メタファーとして）

### 5.2 ビジュアル方針
- **基調**：glitch / ノイズ多め
- 表現要素：
  - スキャンライン、ピクセルソート、RGBずれ
  - テクスチャ崩れ、ポリゴン破綻風2D表現
  - 背景の文字化け、コード片の流れ
  - 一時停止・コマ落ち風の演出（演出として制御）
- カラーパレット：暗背景＋ネオン系（シアン／マゼンタ／ライム）でglitch感を出す
- 主人公機：クリーンで幾何学的なデザイン（バグとの対比）

### 5.3 サウンド
- BGM：チップチューン × ノイズ／グリッチ系
- SE：エラー音、ビープ、データ破損音などをモチーフに

## 6. 敵・ステージ設計（中規模ボリューム）

### 6.1 敵タイプ
- 雑魚バグ：直進／追尾／編隊の3種程度
- 中型バグ：耐久あり、特殊弾を撃つ
- ギミック型：地形バグ（一時的に画面が反転、操作が逆転 等）
- 中ボス：各面に1体
- ステージボス：各面1体、最終ボスは大型

### 6.2 ステージ案（メタファー軸で5〜8面）
1. 入力層 / フロントエンド：チュートリアル兼ねる
2. メモリ領域：浮遊するデータ片が地形に
3. ネットワーク層：パケットが弾幕に
4. ファイルシステム：地形ギミック面
5. カーネル：高速戦
6. （任意）破損領域：ボーナス的な崩壊面
7. （任意）バックアップ層
8. コアダンプ：最終ボス

## 7. UI/UX

- メインメニュー：START / CONTINUE / OPTION / HOW TO PLAY
- HUD：ライフゲージ、ボンバー残数、スコア、ステージ名
- ポーズ：ESCキー
- オプション：音量、操作キー設定、画面シェイク強度（演出強度スライダー推奨）

## 8. 開発スコープ（1〜2ヶ月想定）

| フェーズ | 内容 | 期間目安 |
|---|---|---|
| プロトタイプ | 操作・1面・1ボス・基本glitch表現 | 2週間 |
| 量産 | 残りステージ・敵・ボス | 3〜4週間 |
| 仕上げ | UI、サウンド、調整、バグ修正 | 1〜2週間 |

## 9. アクセシビリティ配慮

glitch多用ゲームなので、以下は必須：
- 「演出強度」設定（画面シェイク・ノイズ・点滅を弱められる）
- 光感受性発作対策の警告表示
- 色だけに依存しないUI（敵弾と自弾の形状を変える等）

## 10. プロジェクト構成

```
debugger-stg/
├── docs/
│   └── REQUIREMENTS.md
├── src/
│   ├── scenes/      (BootScene, TitleScene, GameScene, GameOverScene)
│   ├── entities/    (Player, Enemy, Bullet, Boss)
│   └── systems/     (SpawnSystem, ScoreSystem, GlitchEffectSystem)
├── assets/          (仮素材＝単色矩形でOK、後で差し替え)
├── index.html
├── package.json
└── vite.config.js
```

## 11. 直近のゴール（プロトタイプ第一歩）

1. Vite + Phaser 3 でプロジェクト初期化
2. タイトル画面 → ゲーム画面 → ゲームオーバー画面の遷移
3. ゲーム画面で自機が矢印キーで動き、Zキーで弾を撃てる
4. `npm run dev` で起動できる
